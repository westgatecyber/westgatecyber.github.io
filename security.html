Security

Public key exchange

We have developed a novel certificate-based method of mutual authentication using public key infrastructure (PKI) concepts, and combined it with modern peer-to-peer networking to significantly reduce the friction of deploying secure communication channels between users and systems.

Our exact method of key exchange is currently under NDA, if you would like you know more please contact us.

Cryptography

  • Elliptic curve cryptography for key exchange and key signing

    We use ECDHE cryptography with Curve25519 to exchange keys, and Ed25519 to sign messages. We use the libsodium implementation of NaCl.

  • AES message encryption

    For symmetric encryption we use AEAD_AES_256_CBC_HMAC_SHA_512 mode with PKCS#7 padding.

  • We Encrypt-then-MAC

    Using HMACSHA512 on AES256 in AEAD mode.

  • We implement perfect forward secrecy

    Long term keys for authentication, short term ephemeral keys for each new session.

  • All comparisons are performed in constant-time

    Removes an entire class of side-channel attacks.

  • No support for data compression

    Compression can leak information about the plaintext through side-channel attacks.

  • Invalid packets are dropped, and do not elicit error messages as part of the protocol specification

    Reduces the risk of successful padding oracle attacks.

  • Session resuming is not permitted, nor part of the protocol

    Minimises the attack surface of the protocol to renegotiation attacks.

  • Key rotation is forced after every 2^32 bytes of data exchanged between peers

    Well below the recommended key refresh boundary for AES256 (2^64).

  • A narrower public key infrastructure

    Instant VPN operates with a single CA at any given point in time. In traditional TLS any of 600+ vendors can issue a certificate against your public key. In our PKI model, only one CA is active at any point in time, therefore reducing the risk of an obscure CA you've never heard of being compromised or going rogue and exposing you to man-in-the-middle attacks.
